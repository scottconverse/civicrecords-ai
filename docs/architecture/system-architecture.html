📎 Ingestion Pipeline — Two-Track
Fast Track (Structured)
PDF, DOCX, XLSX, CSV, HTML, email, text → parse → chunk → embed → pgvector
LLM Track (Scanned/Image)
Scanned PDFs, JPEG, PNG, TIFF → Gemma 4 multimodal → text → chunk → embed. Tesseract fallback

Key Data Flows

🔍 Search Query

1
Staff enters natural language query
2
Auth + department scope verified
3
Semantic search via pgvector embeddings
4
Keyword search via PostgreSQL full-text
5
Reciprocal Rank Fusion (0.7/0.3 weighting)
6
Results with source attribution + confidence
7
Optional LLM summary (labeled AI draft)

📄 Document Ingestion

1
Admin configures data source
2
Celery worker picks up task from Redis
3
File routed to Fast Track or LLM Track
4
Sentence-aware chunking with overlap
5
nomic-embed-text generates 768-dim vectors
6
Chunks + embeddings stored in pgvector
7
Status updated, audit log written

🔒 Exemption Detection

1
Staff triggers scan on request
2
State-specific rules loaded (keyword + regex)
3
Each document chunk scanned
4
Matches create flags (status: flagged)
5
Optional LLM secondary pass
6
Staff reviews: accept or reject each flag
7
Rates tracked on auditability dashboard

🏢 Department Scoping NEW

1
JWT auth loads user + department_id
2
require_role() checks minimum role
3
List endpoints: WHERE department_id filter
4
Single-resource: check_department_access()
5
Admin bypasses all department checks
6
Shared resources (NULL dept) visible to all
7
All access decisions audit-logged

Key Architecture Decisions

Single Database

PostgreSQL + pgvector handles everything: app data, vectors, audit logs, user accounts. One backup, one restore, one connection pool.

Model Abstraction

LLM interface abstracts Ollama. Swap models (Gemma 4, Llama, Mistral) without touching app code. Embedding model also swappable.

Federation-Ready API

REST API supports both human users (JWT) and service accounts (API keys). Another CivicRecords AI instance can query this one.

Compliance by Default

Audit logging, human-in-the-loop enforcement, and AI content labeling are in the API layer, not the UI. Can't be bypassed.

Docker Compose Stack (5 services)

1. postgres — PostgreSQL 16 + pgvector (data, vectors, audit)
2. redis — Task queue broker
3. api — FastAPI application server
4. worker — Celery worker(s) for async ingestion/embedding
5. ollama — Local LLM runtime (Gemma 4 + nomic-embed-text)
+ frontend — React build served by api (or nginx in production)
